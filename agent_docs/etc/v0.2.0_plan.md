# 버전 0.2.0 실행 계획 (2026년 2월 9일)

**목표:** 백엔드의 데이터 수집 방식을 체계적인 크롤링 시스템으로 전환하고, 프론트엔드 UI 전체를 '탐험 지도'라는 새로운 컨셉에 맞춰 전면 개편하여 사용자 경험을 극대화합니다.

---

## Part 1: 백엔드 - 대규모 데이터 수집 시스템 구축

기존 `changeplan.md`의 내용을 바탕으로, Genius API의 한계를 극복하고 확장성 있는 데이터 파이프라인을 구축합니다.

-   **핵심 목표:** `requests`와 `BeautifulSoup4`를 사용하여 나무위키와 같은 공개 위키 사이트의 데이터를 수집하는 독립적인 크롤러(`crawler/namu_crawler.py`)를 개발하고 시스템화합니다.

-   **주요 작업:**
    1.  **견고한 파서(Parser) 개발:** 위키의 다양한 페이지 구조(표, 리스트, 문단)에 대응하여 곡, 아티스트, 참여 정보 등을 안정적으로 추출하는 로직을 구현합니다.
    2.  **데이터베이스 연동 로직 수정:**
        -   크롤링한 데이터를 저장할 수 있도록 `models.py`의 `Song`, `Person` 모델에 `source_url`과 같은 출처 컬럼을 추가합니다.
        -   `crud.py`에 크롤러가 직접 호출하여 정제된 데이터를 DB에 저장하는 함수(`create_crawled_song`)를 구현합니다.
    3.  **크롤링 시스템화:**
        -   크롤링 작업을 시작, 중지, 모니터링할 수 있는 관리 스크립트를 작성합니다.
        -   어디까지 탐색했는지 추적하기 위한 큐(Queue)와 방문 목록(Visited List)을 파일 또는 DB 기반으로 관리하여 작업의 연속성을 보장합니다.
        -   서버에 과부하를 주지 않도록 `robots.txt` 준수 및 요청 간 지연(delay) 설정 등 크롤링 윤리를 준수합니다.

## Part 2: 프론트엔드 - '탐험 지도' UI 컨셉 도입

단순한 그래프 뷰를 넘어, 사용자가 관계의 바다를 '탐험'하는 듯한 몰입감 있는 경험을 제공하는 것을 목표로 합니다.

-   **핵심 컨셉:** 모든 UI 요소를 '탐험 지도'라는 하나의 컨셉 아래 일관성 있게 재설계합니다.

-   **추천 작업:**

    1.  **비주얼 컨셉 디자인:**
        -   **지도 스타일 배경:** 단색 배경을 양피지, 고지도, 성좌도, 디지털 회로도 등 특정 테마를 가진 텍스처 배경으로 교체하여 '지도'의 느낌을 강화합니다.
        -   **'영역(Region)'과 '랜드마크(Landmark)':**
            -   단순 나열이 아닌, 장르, 시대, 소속사, 협업 빈도 등 특정 기준에 따라 노드들을 '영역'으로 그룹화합니다.
            -   영향력이 큰 아티스트나 곡(연결이 많은 노드)은 지도 위의 '주요 도시'나 '랜드마크'처럼 더 크고 화려하게 렌더링합니다.
        -   **'경로(Path)'와 '길(Road)':** 엣지(선)를 길이나 항로처럼 스타일링하고, 두께나 색상, 애니메이션으로 관계의 깊이(예: 협업 횟수)를 표현합니다.

    2.  **UI 컴포넌트 재설계:**
        -   **검색창 → '나침반/망원경':** 검색창을 지도 위에서 특정 '위치'를 찾는 도구 컨셉으로 변경합니다. 검색 결과는 해당 노드로 지도가 부드럽게 이동하며 줌인되는 효과를 줍니다.
        -   **사이드 패널 → '탐험 일지/도감':** 노드 선택 시 나타나는 상세 정보 패널을 '탐험 일지'나 '생물 도감'처럼 재디자인합니다. 노드의 정보, 참여 작품, 관련 인물 등을 '탐사 기록'처럼 표시합니다.
        -   **컨트롤 → '지도 도구':** 줌/패닝 컨트롤을 돋보기, 나침반 등 지도 관련 도구 아이콘으로 변경합니다.

    3.  **인터랙션 및 애니메이션 강화:**
        -   **'전장의 안개(Fog of War)':** 처음에는 지도의 일부만 보이고, 사용자가 노드를 클릭하며 탐험함에 따라 주변 영역이 '발견'되며 안개가 걷히는 듯한 효과를 추가하여 탐험의 재미를 부여합니다.
        -   **탐험 애니메이션:** 새로운 노드로 포커스가 이동할 때, 즉시 화면이 바뀌는 대신 지도를 가로질러 '여행'하는 듯한 카메라 이동 애니메이션을 추가합니다.
        -   **LOD(Level of Detail):** 구글맵처럼, 지도를 축소하면 주요 '랜드마크'만 보이고, 확대할수록 세부적인 노드와 '경로'가 나타나도록 하여 대규모 그래프에서의 성능과 시각적 명료성을 동시에 확보합니다.
