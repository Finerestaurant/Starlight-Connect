# 프로젝트 작업 및 기록 가이드라인

이 문서는 프로젝트의 모든 작업을 체계적으로 기록하고 추적하기 위한 워크플로우를 정의합니다.

## 1. 문서 구조

-   `PLAN.md`: 프로젝트의 전반적인 목표와 초기 계획을 담습니다.
-   `PROGRESS.md`: 프로젝트의 주요 마일스톤 및 과거 진행 상황을 **요약**하여 기록합니다. (End-of-day 요약이 누적되는 곳)
-   `future_plan.md`: 앞으로 나아갈 방향과 미래 비전을 기술합니다.
-   `WORKFLOW.md`: 현재 문서. 작업 흐름과 기록 규칙을 정의합니다.
-   `daily/`: 일일 작업 기록을 날짜별 파일로 저장하는 폴더입니다.
-   `monthly/`: 월별 작업 요약을 날짜별 파일로 저장하는 폴더입니다.
-   `etc/`: 기타 참고 자료 및 아카이브된 문서를 보관합니다.

## 2. 일일 작업 프로세스

모든 작업은 **'계획 -> 진행 및 기록 -> 요약'** 의 흐름을 따릅니다. 이를 통해 작업의 연속성을 확보하고, 특정 시점의 고민과 해결 과정을 쉽게 찾아볼 수 있습니다.

### 단계 1: 작업 계획 (Start of Day)

-   **일일 파일 생성**: `daily/` 폴더 아래에 `YYYY-MM-DD.md` 형식으로 오늘의 날짜에 해당하는 새 마크다운 파일을 생성합니다.
-   **계획 수립**: 오늘 진행할 작업의 목표와 계획을 파일에 먼저 기록합니다.

### 단계 2: 작업 진행 및 기록 (During the Day)

-   **자유로운 상세 기록**: 작업을 진행하면서 발생하는 **모든 생각과 과정**을 일일 파일에 자유롭게 기록합니다.
    -   **무엇을**: 코드 스니펫, 에러 메시지, 명령어, 아이디어, 리서치한 링크 등 형식에 구애받지 않고 모든 것을 기록합니다.
    -   **왜**: "왜 이 방법을 선택했는지", "왜 문제가 발생했는지" 등 의사결정의 이유와 고민의 흔적을 남기는 것이 중요합니다.
    -   이 기록은 상세하고 솔직할수록 나중에 큰 도움이 됩니다.

### 단계 3: 작업 요약 (End of Day)

하루의 작업을 마무리할 때, 다음 두 가지를 수행하여 작업 내용을 요약하고 아카이빙합니다.

1.  **그날의 작업 요약**:
    -   `daily/YYYY-MM-DD.md`에 기록된 상세 내용을 바탕으로, 오늘 완료한 핵심 작업을 1~3줄로 간결하게 요약합니다.

2.  **요약 내용 업데이트**:
    -   **월별 기록**: `monthly/` 폴더 아래에 `YYYY-MM.md` 형식의 월별 파일에 날짜와 함께 요약 내용을 추가합니다. 파일이 없으면 새로 생성합니다.
        -   예시: `[YYYY-MM-DD] - 백엔드 크롤링 시스템 구조 설계 완료.`
    -   **전체 진행 상황**: `PROGRESS.md` 파일의 **최상단**에 동일한 요약 내용을 추가합니다. 이를 통해 `PROGRESS.md`는 항상 프로젝트의 최신 주요 변경사항을 보여줍니다.

---

## 3. 예시

**상황**: 2026년 2월 11일, 사용자가 '탐험 지도' UI 컨셉을 구체화하는 작업을 진행.

1.  **파일 생성**: `daily/2026-02-11.md` 파일을 만들고 아래와 같이 기록합니다.

    > ## '탐험 지도' UI 컨셉 구체화
    >
    > -   피그마로 기본 와이어프레임 스케치
    > -   '전장의 안개' 효과를 구현할 라이브러리 리서치 (react-fog, css mask 등)
    > -   지도 배경 텍스처 소스 탐색 (unsplash, pexels)
    > -   ... (기타 생각들)

2.  **작업 종료 및 요약**:

    -   **요약**: `'탐험 지도' UI 컨셉 와이어프레임 설계 및 '전장의 안개' 효과 리서치 진행.`

3.  **업데이트**:
    -   `monthly/2026-02.md` 파일에 다음 내용을 추가:
        > -   `[2026-02-11]` - '탐험 지도' UI 컨셉 와이어프레임 설계 및 '전장의 안개' 효과 리서치 진행.
    -   `PROGRESS.md` 파일 최상단에 다음 내용을 추가:
        > ## 2026년 2월 11일
        >
        > -   '탐험 지도' UI 컨셉 와이어프레임 설계 및 '전장의 안개' 효과 리서치 진행.
